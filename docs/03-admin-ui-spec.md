# FlowChat - Admin UI Specification

## Design Principles

1. **Progressive Disclosure**: Show simple options first, reveal complexity on demand
2. **Live Preview**: Every change visible immediately
3. **Contextual Help**: Tooltips and inline documentation
4. **Mobile-Friendly**: Admin UI works on tablets
5. **Accessibility**: WCAG 2.1 AA compliant

## Responsive Behavior

### Desktop (1200px+)
- Full three-column layout (sidebar + editor + preview)
- Preview always visible

### Tablet (768px - 1199px)
- Two-column layout (sidebar + editor)
- Preview in modal/drawer

### Mobile (< 768px)
- Single column, stacked layout
- Tabs become accordion
- Preview in full-screen modal

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Ctrl/Cmd + S` | Save changes |
| `Ctrl/Cmd + P` | Toggle preview |
| `Ctrl/Cmd + /` | Toggle simple/advanced mode |
| `Escape` | Close modal |
| `Tab` | Navigate between fields |

## Accessibility Requirements

- All form inputs must have associated labels
- Color contrast ratio minimum 4.5:1
- Focus indicators visible on all interactive elements
- Error messages announced to screen readers
- Modal focus trap when open
- Keyboard navigation for all features
